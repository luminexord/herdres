# Security

Herdres is a local Telegram connector for Herdr/Tendwire workflows. It assumes a
trusted local machine and trusted service account. Do not expose Herdres command
entry points, state files, lock files, or environment files to untrusted users.

## Source Mode Boundary

In `HERDRES_TENDWIRE_MODE=source`, Herdres owns Telegram mapping, formatting,
buttons, topic/message state, delivery retries, and rate limits. Tendwire owns
Herdr observation, private bindings, turns/pending interactions, attention,
command routing, receipts, backend health, event/projection state, and connector
outbox.

Normal source-mode behavior must not call direct Herdr pane APIs such as
`pane_list`, `pane_by_id`, `pane_turn`, `prefetch_pane_turns`, `send_to_pane`,
`herdr pane send-keys`, or `herdr pane read`. Use `off` or `enrich` only as
explicit rollback/debug modes.

## Secrets And State

Never commit `herdres.env`, bot tokens, Telegram chat/topic/message IDs, local
state files, lock files, session files, logs, caches, or generated bytecode.
`.env.example` is a template only.

Tendwire public JSON must not expose raw `pane_id`, `terminal_id`,
`backend_target`, raw target values, private fingerprints, Telegram IDs, socket
paths, argv/env/stdout/stderr, tokens, or secrets. Herdres source mode should use
Tendwire worker IDs and fingerprints only as public routing metadata.

## Reporting

When sharing bug reports, include sanitized JSON output from `herdres doctor`,
`herdres tendwire source-smoke`, and relevant test commands. Redact tokens,
local usernames, private paths, Telegram IDs, and raw pane or terminal IDs.
